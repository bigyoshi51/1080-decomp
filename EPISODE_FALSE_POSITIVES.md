# False-Positive Episode Audit (INCLUDE_ASM tautology)

Generated 2026-05-16 during a source-5 /decompile tick.

## Problem

140 episodes were logged on `#ifdef NON_MATCHING / #else INCLUDE_ASM`
wraps whose C body is <100%% and which have NO post-cc recipe
(INSN_PATCH/SUFFIX/PREFIX/PROLOGUE) routing the real build through the C.
Their episode text explicitly states match via "INCLUDE_ASM tautology" /
"fuzzy<100 reflects partial C-decomp" and records final_match_percent=100.

These violate the core invariant (CLAUDE.md): *"Episodes are for exact
matches only. NON_MATCHING wraps don't get episodes — they'd train on
wrong bytes."* The build path is INCLUDE_ASM (the .s bytes ARE the ROM
bytes), so byte-equality is tautological; the C the episode captures only
~19-78%% reproduces the target. Feeding these to SFT/verifier-RL
(TRAINING_PLAN.md) trains "this partial C == exact match".

Far exceeds the "~7" in `feedback_land_script_byte_verify_regression.md`.

## Remediation

Per the invariant these episodes should be REMOVED (git-recoverable).
This audit tick removed only the one surfaced by the source-5 roll
(`game_uso_func_00007538`, C=48.97%%). The remaining 139 are
listed below for a deliberate bulk decision (high blast radius on shared
training data — flagged to the user, not silently mass-deleted).

Re-generate/verify this list:
`grep -rl 'INCLUDE_ASM tautology' episodes/` then drop any func with a
`<func>=0x` recipe in the Makefile.
